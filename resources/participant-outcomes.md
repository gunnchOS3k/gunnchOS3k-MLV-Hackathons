# Participant Outcomes — Zero-to-Hero Build Sprint

**Walk In Curious. Walk Out Portfolio Ready.**

This guide applies to **gunnchOS3k MLV Hack Days** — planned Summer 2026, beginner-friendly, phone-first community build events.

---

## The transformation framing

- **Zero-to-Hero Build Sprint**
- **Hyperbolic Time Machine Experience** — a short, intense stretch where you learn fast by building real things
- **Participants walk in as curious learners and leave as documented builders.**

This event is designed as a **zero-to-hero build sprint**. Participants are not expected to arrive as experts. They are expected to arrive with **curiosity, courage, and willingness to try**. Through rapid problem solving, design thinking, prototyping, documentation, and presentation, participants will leave with more than a memory — they will leave with **proof of work**.

*Nothing here guarantees jobs, prizes, scholarships, or professional status. These outcomes are about **skills and artifacts you earn through participation**.*

---

## What participants practice

| Skill | What it looks like |
|-------|-------------------|
| **Presenting** | Explaining your project to technical and nontechnical audiences |
| **Rapid problem solving** | Making decisions under time constraints |
| **Design thinking** | Empathize → define → ideate → prototype → test |
| **Prototyping** | Building lightweight prototypes on phones, tablets, laptops, or borrowed devices |
| **Slides** | Creating PowerPoint or Google Slides presentations |
| **Documentation** | Writing technical or plain-language project docs |
| **README writing** | One-page explanation anyone can follow |
| **Publishing** | GitHub, GitHub Pages, Devpost, itch.io, or a personal website (when ready) |
| **LaTeX report** | Optional advanced technical write-up |
| **Reproducibility** | Checklist so someone else could understand or rebuild your idea |
| **Demo video** | Short recording showing what you made |
| **Portfolio building** | Turning the project into a resume or LinkedIn artifact |

**Beginners are not required to complete every item.** Start where you are.

---

## The Proof of Work Packet

Every participant or team should leave with a **Proof of Work Packet**. It does not need to be perfect. **It needs to exist.**

### Minimum packet (beginner-friendly)

- [ ] Project title  
- [ ] Problem statement  
- [ ] Who it helps  
- [ ] What was built  
- [ ] Tools used  
- [ ] Screenshots or photos  
- [ ] Demo video or live demo  
- [ ] Slides (Google Slides or PowerPoint)  
- [ ] README or one-page explanation  
- [ ] What we learned  
- [ ] What we would build next  

### Fully loaded packet (aspirational)

- [ ] Public GitHub repo or portfolio page  
- [ ] GitHub Pages or personal website project page  
- [ ] PowerPoint / Google Slides pitch deck  
- [ ] Technical README  
- [ ] LaTeX-style technical report (optional)  
- [ ] Reproducibility checklist  
- [ ] Demo video  
- [ ] Screenshots / GIFs  
- [ ] Asset and license credits  
- [ ] Resume / LinkedIn bullet  
- [ ] Reflection: *“Before this event, I thought I could not… Now I can…”*  

The **fully loaded packet is a goal**, not a requirement for beginners.

## Outcome Badge Ladder

Climb at your own pace. Every level counts.

| Level | Badge | You… |
|-------|-------|------|
| 1 | **I Showed Up** | Attended, joined a team, learned the tools |
| 2 | **I Built Something** | Created a prototype, mockup, game, app, story, or demo |
| 3 | **I Explained It** | Presented to a mixed technical/nontechnical audience |
| 4 | **I Documented It** | Created slides and a README or project explanation |
| 5 | **I Published It** | Posted to GitHub, GitHub Pages, itch.io, Devpost, or a personal site |
| 6 | **I Made It Reproducible** | Added setup steps, tool list, assets, credits, and how-to-run/use notes |
| 7 | **I Became Portfolio Ready** | Left with a project page, demo video, and resume bullet |

> **A phone-only beginner can still complete the ladder.** A clear idea, a working prototype, a short video, and strong documentation can be a real portfolio artifact.

---

## Pressure with guardrails

This is meant to be a **positive pressure-cooker experience**, not burnout or humiliation. The goal is to stretch participants with time constraints, teamwork, demos, and public explanation — while still providing mentorship, beginner tracks, phone-first tools, youth safety rules, and encouragement.

> **The pressure is the mold, but support is the safety rail.**
