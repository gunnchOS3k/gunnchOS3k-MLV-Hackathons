# Volunteer and Mentor Guide

**Planned Summer 2026 · Volunteer interest form coming soon**

Community hack days run on caring volunteers. Thank you for considering a role.

---

## Volunteer roles

| Role | What you do |
|------|-------------|
| **Check-in helper** | Welcome participants, distribute materials, answer first questions |
| **Room guide** | Help people find workspaces, schedule blocks, and facilities |
| **Phone-first tech helper** | Support browser-based Google tools, Replit web, mobile demos |
| **Beginner mentor** | Encourage first-time hackers; keep conversations **open and visible** |
| **Pitch coach** | Help teams explain their idea clearly in slides and demos |
| **Demo reviewer** | Friendly practice feedback before judging |
| **Food/snack helper** | Support refreshments if sponsors confirm (not guaranteed until confirmed) |
| **Safety/chaperone support** | Help maintain appropriate supervision for youth participants |
| **Photographer/media** | Only after media release forms are in place |
| **Judge** | Score projects using inclusive, beginner-friendly criteria |

---

## Mentor expectations

1. **Be encouraging** — many participants have never coded  
2. **Stay visible** — no one-on-one private mentoring with minors  
3. **Phone-first mindset** — do not assume everyone has a laptop  
4. **Ask questions** — help teams clarify problem, user, and demo  
5. **Follow Code of Conduct** — model respect and inclusion  

---

## Phone-first mentoring tips

- Suggest **Google Docs** for planning before jumping to code  
- Help record a **short demo video** early so teams know their deadline  
- Celebrate **clear explanations** over complicated tech  
- Point to [Phone-First Toolkit](phone-first-toolkit.md)
