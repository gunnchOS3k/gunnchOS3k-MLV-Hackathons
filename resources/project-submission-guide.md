# Project Submission Guide

**Planned event · Registration and final submission platform coming after MLH/venue confirmation**

Final official instructions may use **MLH Challenges** and/or **Devpost** depending on event setup. This guide helps teams prepare now in beginner-friendly language.

---

## Before you submit

Make sure your demo answers:

1. **What problem are you solving?**  
2. **Who does it help?**  
3. **What did you actually build?**  
4. **How can a judge see it in 1–3 minutes?**

> **No shame:** A polished idea with a clear demo can beat a complicated project nobody understands.

---

## Beginner submission checklist

- [ ] **Project name**  
- [ ] **Team members** (first names or display names — follow privacy rules)  
- [ ] **Short problem statement** (1–3 sentences)  
- [ ] **Who the project helps**  
- [ ] **What was built** (plain language)  
- [ ] **Link** to Google Doc, Slides, Replit, GitHub, Canva, Drive folder, etc.  
- [ ] **Demo** — 1–3 minute video link or live demo plan  
- [ ] **What the team learned**  
- [ ] **What you would build next**  
- [ ] **Track** (optional): Community, AI, Game, Design, Web, Beginner  

---

## Advanced submission checklist

Everything in the beginner list, plus:

- [ ] Public **GitHub** repository  
- [ ] **README** with project overview  
- [ ] **Setup instructions** (how to run or view)  
- [ ] **Demo link** (hosted app or live URL)  
- [ ] **Demo video**  
- [ ] **Open-source license** (if appropriate)  
- [ ] **Technical explanation** — architecture, APIs, data sources  

---

## Demo video tips (phone-friendly)

1. Introduce your team and project name (10 seconds)  
2. State the problem and who it helps (20 seconds)  
3. Show the prototype — scroll through Doc, play game, open Replit, etc. (1–2 minutes)  
4. Say what you learned and what is next (20 seconds)  

Record in a quiet space. Landscape mode often works better for slides and apps.

---

## Privacy and safety

Do **not** include in public submissions:

- Passwords or API keys  
- Home addresses  
- Private school IDs  
- Medical details  
- Other sensitive personal information
